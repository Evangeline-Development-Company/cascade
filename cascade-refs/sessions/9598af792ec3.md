# Design Document: Valhalla Capital — The "Bartender's Interface" Evolution

## 1. Vision & Core Philosophy
The current **Valhalla Capital** dashboard is a functional, aesthetically pleasing monitoring tool inspired by *VA-11 Hall-A: Cyberpunk Bartender Action*. To take it from "functional" to "complete," we need to deepen the immersion by transforming data into a narrative. The dashboard shouldn't just show *what* the bot is doing; it should show the *personality* of the fund.

**Design Pillar:** *Every piece of data is an ingredient in a cocktail.*

---

## 2. Architectural Enhancements

### 2.1 The "Drink Menu" (Asset Allocation)
Instead of a standard `<table>` for positions, we introduce a **Drink Menu** view.
- **Contract:** Assets are mapped to "Ingredients" (e.g., Base Spirits, Mixers, Garnishes).
- **Data Flow:** The backend categorizes holdings by risk/asset class.
- **UI Component:** A stylized vertical menu.
  - *Example:* "BTC (Bourbon) — 45% (The Base). ETH (Sweet Vermouth) — 20% (The Body)."
- **Rationale:** Highers UX engagement by framing abstract financial data through a familiar, thematic lens.

### 2.2 The "Bar Tab" (PnL Tracking)
Performance metrics are often overwhelming. We'll consolidate them into a **Bar Tab** component.
- **Contract:** A new endpoint `/api/performance/tab` returns a JSON structure of daily/weekly/total performance.
- **UI Component:** A fixed-width, monospace, vertical container with a "receipt" aesthetic (dotted borders, thermal print-style typography).
- **Visuals:** Use the existing `pink` (86'd) for negative and `cyan` (Served) for positive.

### 2.3 Valkyrie's "Neural Link" (State Machine)
Moving beyond flat logs, the **Valkyrie** page should feature a "State Visualization."
- **States:** `IDLE`, `SCANNING`, `EVALUATING`, `EXECUTING`, `COOLDOWN`.
- **UI Component:** A neon-lit SVG flow diagram. When Valkyrie is in a specific state, that node glows with a pulse animation.
- **Interaction:** Clicking a node filters the logs below to show only the relevant entries for that state.

---

## 3. Interaction Patterns & UX

### 3.1 "Order a Drink" (Manual Directives)
If the system allows for manual overrides or signal triggering, we use the "Ordering" metaphor.
- **Pattern:** A floating action button (FAB) or a specific "Bar" tab.
- **Interaction Flow:**
  1. User selects a "Drink" (e.g., *The Short Squeeze*).
  2. A confirmation modal appears (styled as a customer order ticket).
  3. On confirmation, an HTMX request triggers the backend trade executor.
- **Safety:** Each "Drink" has a predefined risk profile (The "ABV" - Asset-Based Volatility).

### 3.2 The "Rumor Mill" (Sentiment Feed)
A secondary ticker or a "Chat" box in the corner showing "Bar Talk."
- **Content:** Aggregated headlines or internal bot sentiment (e.g., "Valkyrie is feeling bullish on $SOL today").
- **UI:** Speech bubbles or a scrolling "Whisper" feed that appears/disappears intermittently.

---

## 4. Visual Identity & Aesthetic Polish

### 4.1 Enhanced CRT Effects
- **Scanline Toggle:** Add a small "Power" switch in the footer to toggle the `::after` scanline and flicker effects (Accessibility/UX).
- **Glitch Transitions:** Apply the `@keyframes glitch` to page transitions when HTMX swaps content.
- **Soundscape:** Subtle, non-intrusive 8-bit "blips" on successful HTMX swaps or trade notifications.

### 4.2 Color Mapping (Semantic)
Refining the use of the existing palette:
- **Cyan (`--served`):** Growth, entry orders, healthy connection.
- **Pink (`--eighty-sixed`):** Drawdown, exit orders, connection lost.
- **Amber (`--mixing`):** Pending orders, evaluation in progress.
- **Lilac (`--lilac`):** Strategic pivots, meta-data, secondary assets.
